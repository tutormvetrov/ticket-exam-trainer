# Быстрый старт: подготовка к письменному госэкзамену

Этот сценарий для студентов, у которых есть один большой сборник билетов к письменному госэкзамену в `DOCX` или `PDF` и которые хотят готовиться вместе.

## Что надо один раз

1. Поставить Python 3.12+.
2. `python -m pip install -r requirements.txt`
3. Поставить локальный Ollama (https://ollama.com) и вытянуть модель:
   ```powershell
   ollama pull qwen3:8b
   ```
4. Запустить приложение:
   ```powershell
   python main.py
   ```
5. `Настройки -> Ollama -> Проверить соединение`. Должно быть зелёно и с временем отклика.

## Шаг 1. Импорт сборника билетов как «Госэкзамен»

`Импорт документов -> выбрать файл`.

После парсинга в summary выберите профиль ответа **`Госэкзамен`** (а не `Обычный билет`).
Тогда для каждого билета система построит 6 блоков:

- введение
- теория
- практика
- навыки
- вывод
- дополнительные элементы

Эти блоки — скелет письменного ответа, по которому будет идти scoring.

## Шаг 2. Тренировка в режиме «Полный госответ»

`Тренировка -> выбрать билет из очереди -> режим «Полный госответ»`.

Режим:
- показывает структуру ожидаемого ответа по блокам;
- принимает ваш полный письменный ответ (по одному полю на блок);
- после отправки считает отдельный score по каждому блоку и по критериям;
- попадает в общий профиль освоения.

## Шаг 3. Статистика готовности к госэкзамену

`Статистика` показывает отдельный блок «Профиль готовности к госэкзамену»:
- средний балл по каждому из 6 блоков;
- слабые блоки (куда тратить время перед экзаменом);
- последние сессии именно по гос-профилю.

## Шаг 4. Готовиться вместе

Рекомендуемый маршрут для группы:
1. Договоритесь об одном сборнике билетов, пройдите Шаг 1 синхронно.
2. База данных локальная (`app_data/exam_trainer.db`), у каждого своя — можно обмениваться только сборниками, не результатами.
3. Сверяйтесь по «слабым блокам» в статистике — это даёт точечные темы для общих обсуждений.

## Если что-то не работает

- `Настройки -> Ollama` → `Проверить соединение` — единственный честный индикатор;
- `python -m pytest -q` из корня репозитория должен выдать `PASSED` без `FAILED`;
- при проблемах импорта проверьте, что файл реально `.docx` или `.pdf` и не битый.
