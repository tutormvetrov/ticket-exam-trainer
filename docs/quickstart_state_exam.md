# Быстрый старт: письменный госэкзамен

Сценарий для студента, у которого есть большой сборник билетов в `DOCX` или `PDF` и который хочет готовиться по структуре госэкзамена.

## Что нужно один раз

1. Установить `Python 3.12+`.
2. Выполнить `python -m pip install -r requirements.txt`.
3. Поднять локальный `Ollama` и загрузить модель:

```powershell
ollama pull qwen3:8b
```

4. Запустить приложение:

```powershell
python main.py
```

5. Открыть `Настройки -> Ollama -> Проверить соединение`.

## Шаг 1. Импортируйте сборник как `Госэкзамен`

Откройте `Импорт документов`, выберите файл и в summary задайте профиль ответа `Госэкзамен`.

Тогда для каждого билета система построит 6 блоков:

- введение
- теория
- практика
- навыки
- вывод
- дополнительные элементы

## Шаг 2. Используйте режим `Полный госответ`

Откройте `Тренировку`, выберите билет и включите режим `Полный госответ`.

Он:

- показывает структуру ответа по блокам;
- принимает развёрнутый ответ по частям;
- считает отдельный score по блокам и критериям;
- сохраняет результат в общий профиль освоения.

## Шаг 3. Используйте `Рецензию ответа`

После нескольких полных прогонов можно переключиться в режим `Рецензия ответа`.

Он:

- принимает полный письменный ответ;
- строит потезисный review verdict;
- помогает увидеть слабые места до следующего полного прогона.

## Шаг 4. Смотрите статистику

В `Статистике` доступны:

- средний балл по блокам;
- слабые блоки;
- последние сессии по гос-профилю;
- criterion scores и block scores.

## Шаг 5. Работайте в группе

Практичный маршрут:

1. договориться об одном сборнике;
2. импортировать его локально каждому участнику;
3. обсуждать не чужую БД, а слабые блоки и формулировки ответов;
4. использовать `Рецензию ответа` как второй проход после `Полного госответа`.

## Если что-то не работает

- проверьте `Настройки -> Ollama`;
- убедитесь, что документ реально `DOCX` или `PDF`;
- прогоните `python -m pytest -q`;
- сверяйтесь с текущим ориентиром репозитория: `186 passed, 5 skipped`.
