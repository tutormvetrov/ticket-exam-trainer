# Product Spec

## Назначение

Приложение помогает готовиться к вузовским экзаменам локально. Базовая единица работы здесь не просто билет как текст, а билет как структура знаний, weak areas, exercise templates и adaptive queue.

## Что уже реализовано

1. Импорт материалов из `DOCX` и `PDF` в основном exam-flow.
2. Превращение исходного текста в структурированные билеты и карту знаний.
3. Локальная интеграция с `Ollama`.
4. Adaptive review и scoring по микронавыкам.
5. Профиль `Госэкзамен` с block scores и criterion scores.
6. Отдельный `Dialogue` flow.
7. Отдельный DLC `Подготовка к защите`.
8. Warm-minimal UI shell и редакционный редизайн ключевых экранов.

## Import scope

Основной продукт:

- `DOCX`
- `PDF`

Defense DLC:

- `DOCX`
- `PDF`
- `PPTX`
- `TXT`
- `MD`

## Dialogue scope

Dialogue должен:

- стартовать от конкретного билета;
- использовать grounded ticket context;
- хранить transcript и итог;
- не открываться как fake-screen без подтверждённого Ollama.

## DLC scope

Текущий scope `Подготовки к защите`:

- paywall и локальная активация;
- проекты;
- импорт материалов;
- dossier;
- outline;
- storyboard;
- logical gaps;
- mock defense;
- repair queue.

## Ограничения

1. Никаких облачных LLM.
2. Никаких декоративных статусов подключения.
3. Никакого немаркированного demo-behavior.
4. Никакого ручного редактирования `dist/`.
5. Любой неполный статус должен описываться как неполный.
