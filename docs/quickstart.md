# Быстрый старт

Этот сценарий рассчитан на пользователя Windows, который хочет быстро пройти путь `Ollama -> импорт -> библиотека -> тренировка -> статистика`.

## 1. Запустите приложение

Готовый release:

```text
Tezis\Tezis.exe
```

Из исходников:

```powershell
python -m pip install -r requirements.txt
python main.py
```

## 2. Проверьте локальный Ollama

Откройте `Настройки -> Ollama`.

Что сделать:

1. Нажать `Проверить соединение`.
2. Если Ollama ещё не готов, нажать `Автонастройка Ollama`.
3. После завершения снова нажать `Проверить соединение`.

Система готова, когда:

- endpoint отвечает;
- найдена `qwen3:8b` или совместимая локальная `Qwen`-модель;
- показано время отклика;
- статус подтверждается диагностикой, а не только цветом.

## 3. Импортируйте материалы

Откройте `Импорт документов`.

Основной сценарий:

- один большой `DOCX`

Также поддерживается:

- `PDF`

Что важно:

- отдельного `import preview` пока нет;
- импорт идёт по стадиям с progress и summary;
- после завершения доступны handoff-кнопки в `Библиотеку`, `Тренировку` и `Статистику`.

## 4. Проверьте библиотеку и билеты

Откройте `Библиотеку` и `Билеты`.

Что проверить:

- документ появился в списке;
- видны разделы и билеты;
- карточки билетов и метрики не пустые;
- можно открыть структуру билета и профиль ответа.

## 5. Перейдите к тренировке

Откройте `Тренировку`.

Сейчас в UI доступны 8 режимов:

- `reading`
- `active-recall`
- `cloze`
- `matching`
- `plan`
- `mini-exam`
- `state-exam-full`
- `review`

Минимальный рабочий цикл:

1. выбрать билет из очереди;
2. выбрать режим;
3. дать ответ;
4. получить feedback или review verdict;
5. перейти к следующему билету.

## 6. Используйте дополнительные экраны

- `Диалог` для устной репетиции по билету;
- `Статистика` для результатов и weak areas;
- `Карта знаний` для связей между билетами;
- `Подготовка к защите` для DLC workflow.

## 7. Если готовитесь к письменному госэкзамену

Импортируйте документ с профилем `Госэкзамен`, затем используйте режим `Полный госответ`. Для этого режима система хранит блоки ответа и отдельную статистику по ним.

## 8. Если что-то не работает

Проверка Ollama:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\check_ollama.ps1
```

Полная автоподготовка на Windows:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup_ollama_windows.ps1
```

Базовый тестовый контур:

```powershell
python -m pytest -q
```

Актуальный ориентир для репозитория на 2026-04-17:

- `186 passed, 5 skipped`

## 9. macOS

Код, скрипты и инструкция есть в репозитории, но живой runtime smoke нужно выполнять на реальном Mac.

```bash
python3 main.py
bash scripts/setup_ollama_macos.sh
bash scripts/check_ollama_macos.sh
bash scripts/build_mac_app.sh
```
