# Руководство пользователя

Это руководство описывает текущее состояние продукта на `main` после warm-minimal refresh от 2026-04-17.

## Навигация

В левой панели доступны:

- `Библиотека`
- `Предметы`
- `Разделы`
- `Билеты`
- `Импорт`
- `Тренировка`
- `Диалог`
- `Статистика`
- `Карта знаний`
- `Подготовка к защите`
- `Настройки`

Верхняя панель показывает актуальные `title` и `subtitle` выбранного раздела.

## Настройки и Ollama

Откройте `Настройки -> Ollama`.

Здесь можно:

- проверить локальный endpoint;
- выбрать модель;
- обновить список моделей;
- запустить автоподготовку Ollama;
- открыть служебные скрипты и документацию.

Рабочим статус считается только тогда, когда диагностика показывает живой endpoint, модель и время отклика.

## Импорт

Экран `Импорт` запускает основной ingest-flow.

Основной продукт поддерживает:

- `DOCX`
- `PDF`

Defense DLC дополнительно поддерживает:

- `PPTX`
- `TXT`
- `MD`

После успешного импорта приложение показывает summary и handoff в другие разделы.

## Библиотека

`Библиотека` показывает:

- список документов;
- детали выбранного документа;
- быстрые переходы в тренировочные режимы;
- readiness и сервисные подсказки.

После warm-minimal refresh это главная editorial-витрина продукта.

## Предметы и разделы

`Предметы` и `Разделы` дают сводку по структуре импортированного материала и позволяют быстро сузить выбор для дальнейшей работы.

## Билеты

`Билеты` показывают карточки билетов, answer profile, weak areas и связанные элементы карты знаний. Это экран для проверки того, что импорт дал не просто текст, а полезную структуру.

## Тренировка

В `Тренировке` сейчас доступны 8 отдельных режимов:

- `reading`
- `active-recall`
- `cloze`
- `matching`
- `plan`
- `mini-exam`
- `state-exam-full`
- `review`

Что делает каждый режим:

- `reading`: чтение билета и эталонного ответа;
- `active-recall`: воспроизведение по памяти до подсказки;
- `cloze`: заполнение смысловых пропусков;
- `matching`: сопоставление терминов и определений;
- `plan`: сбор структуры ответа;
- `mini-exam`: короткий экзаменационный прогон;
- `state-exam-full`: развёрнутый ответ по блокам госэкзамена;
- `review`: потезисная рецензия полного ответа.

## Диалог

`Диалог` нужен для устной репетиции по билету. Он использует ticket-grounded context, сохраняет сессии и блокирует запуск, пока Ollama не подтверждён.

## Статистика

`Статистика` показывает:

- средний результат;
- weak areas;
- recent sessions;
- профиль готовности для госэкзамена, если использовался соответствующий answer profile.

## Карта знаний

`Карта знаний` визуализирует связи между билетами и помогает видеть не только список тем, но и пересечения материала.

## Подготовка к защите

Это отдельный DLC workspace. Что уже работает:

- paywall;
- локальная активация по ключу;
- создание проекта;
- импорт материалов;
- dossier;
- outline;
- storyboard;
- logical gaps;
- mock defense;
- repair queue.

## Что важно помнить

- продукт локальный, без облачных LLM;
- `dist/` не редактируется вручную;
- import preview до старта импорта пока не реализован;
- macOS runtime smoke остаётся отдельным QA-пунктом.
