# VM Smoke Checklist — Tezis v2.0 release verification

**Когда запускать:** перед рассылкой `Tezis-v2.0-windows.zip` однокурсникам (ориентировочно 10-11 мая 2026).
**Цель:** поймать проблемы, которые видны только на чистой Windows-машине: SmartScreen, отсутствующие шрифты, Ollama install, file associations, DPI-scaling.
**Время на проход:** ~40 минут на одну VM, ~90 минут на две (10 + 11).

---

## Предусловия

- Две VM (или два физических ноутбука): **Windows 10 22H2** и **Windows 11 23H2+**. Лучше чистая установка без Visual Studio / Python / Ollama — имитирует классмейта.
- Заранее подготовленный `Tezis-v2.0-windows.zip` с `Tezis.exe`, `data/state_exam_public_admin_demo.db`, `scripts/install_ollama_wizard.ps1`, `README_classmates.md`.
- Доступ к интернету (для Ollama pull, первого запуска Flet если шрифты не забандлены).
- Разрешение по умолчанию (1920×1080). Дополнительно проверить 1366×768 через изменение resolution в Windows Settings → System → Display.

## Checklist

### 1. Распаковка и запуск

- [ ] Скачать `Tezis-v2.0-windows.zip` в `Downloads`.
- [ ] Распаковать через штатный Windows Explorer (не 7-Zip), чтобы убедиться что ZIP не содержит путей с кириллицей, которые Explorer режет.
- [ ] Двойной клик на `Tezis.exe`.
  - **Ожидается:** появляется SmartScreen «Windows protected your PC». Нажать «More info» → «Run anyway». Это нормально для неподписанных exe; в README это шаг 3. Запомни: если SmartScreen молчит — возможно, AV уже блокировал .exe молча.
  - Записать: **да / нет** SmartScreen появился.
- [ ] Приложение открывается **в полноэкранном режиме**. Проверить: видно главный экран «Билеты», 208 записей в списке.

### 2. Fullscreen и window mode

- [ ] Нажать `Esc` — приложение переходит в оконный режим (размер ≈ 1440×900, если не был ранее изменён).
- [ ] Настройки → Окно → «Полноэкранный» — обратно разворачивается во весь экран.
- [ ] Закрыть приложение. Перезапустить. **Ожидается:** открывается в том режиме, в котором его оставили (persistence через `settings.json`).

### 3. Responsive layout

- [ ] На **1920×1080** — видны 2 колонки (список + деталь); правый FSRS-pane может появиться если экран >=1920 в Windows Settings.
- [ ] На **1366×768** (изменить в Windows Settings → Display → Display resolution):
  - Приложение адаптируется: chip-навигация сворачивается, карточки вертикальным списком.
  - Текст читаем, нет двойного скролла, нет обрезания.
- [ ] На **2560×1440** (если VM поддерживает): три колонки с правым FSRS-pane. Видно «Сегодня к повторению: N».

### 4. Ticket navigation + training modes

- [ ] Выбрать любой билет (например, первый в списке) — справа появляется деталь с `canonical_answer_summary`, разделом, лектором.
- [ ] Кнопка «Тренировать» → выбрать режим `reading` → открывается экран Training с полным текстом ответа.
- [ ] `plan` — показывает 6 блоков, кнопки ↑/↓ работают, «Проверить» возвращает позиции.
- [ ] `cloze` — текст с пропусками, поля TextField, проверка.
- [ ] `active-recall` — большое поле ввода + таймер. Написать любой короткий ответ, «Проверить».
  - **Ожидается без Ollama:** возвращается score + weak_points + плашка «Упрощённая рецензия». Время ответа < 3 сек.
  - **С Ollama:** возвращается review verdict + strengths + recommendations. Время 30-90 сек.
- [ ] `state-exam-full` — 6 TextField'ов + count-down таймер с пресетами 20/30/40 минут. Таймер стартует и тикает.
- [ ] `review` — поле для полного ответа + ReviewVerdict widget.

### 5. Ollama install flow (опционально — только если тестируем с LLM)

- [ ] `Win+X` → «Windows PowerShell (Admin)». `cd` в распакованную папку Tezis.
- [ ] Запустить `.\scripts\install_ollama_wizard.ps1`. В первом проходе `-DryRun` не использовать — нам нужен реальный install.
  - **Ожидается:** hardware detect (RAM, CPU cores, VRAM), рекомендация тира, установка Ollama через `winget install Ollama.Ollama` (или через fallback-скачивание .exe).
  - Подтвердить licenses.
  - `ollama pull qwen2.5:7b-instruct-q4_K_M` — должно занять 5-20 минут в зависимости от интернета (~4.7 GB).
  - Canary test — модель отвечает «да» на промпт.
  - `%LOCALAPPDATA%\Tezis\app_data\settings.json` обновлён — видно `"model": "qwen2.5:7b-instruct-q4_K_M"`.
- [ ] Перезапустить Tezis. В TopBar / Settings бейдж Ollama должен стать **зелёным** (Ollama OK · qwen2.5:7b).
- [ ] Открыть билет → `active-recall` → написать ответ → «Проверить». Должна прийти содержательная рецензия с per-thesis verdicts (не keyword-fallback).

### 6. FSRS persistence

- [ ] Выполнить `state-exam-full` на билете №1 со score ≥60. После сохранения attempt — в правой панели Tickets view появляется этот билет в очереди с «due через 1 день» (cold-start ladder step 1).
- [ ] Перезапустить приложение. Очередь должна **сохраниться** (SQLite persistence работает).
- [ ] Повторить для билета №2, №3 — видим что «Сегодня к повторению» растёт.

### 7. Data persistence и upgrade path

- [ ] `%LOCALAPPDATA%\Tezis\app_data\exam_trainer.db` создан после первого запуска.
- [ ] Закрыть Tezis, удалить этот файл, запустить заново — БД пересоздаётся с seed из `data/` папки zip'а. Данные предыдущих attempt'ов теряются (это ожидаемо).
- [ ] Скопировать `app_data/` на другой компьютер с тем же Tezis — приложение видит все прошлые attempt'ы.

### 8. Theme switching

- [ ] Settings → Тема → «Тёмная» — цвета меняются без перезапуска. Accent становится десатурированным rust-ом.
- [ ] Обратно в «Светлая» — возврат к parchment.

### 9. Anti-crash sanity

- [ ] Поиск по заголовку — ввести кириллицу («Конституция», «Федер»). Фильтр срабатывает.
- [ ] Выбрать билет с `source_missing_in_conspect` (отмечен иконкой) — у него 49 из 208. Открыть `reading` — видно сгенерированный ответ без артефактов.
- [ ] Escape во время таймера `state-exam-full` — таймер не падает (продолжает тикать в оконном режиме).
- [ ] Закрыть экзамен-full на полувыполненном ответе — система сохраняет drafted-data или мягко теряет (проверить, что нет краша).

### 10. Производительность на слабом железе

- [ ] (Если VM это имитирует): на машине с 8 GB RAM без GPU приложение стартует < 5 сек, смена route < 0.5 сек, поиск по 208 билетам мгновенный. Pyinstaller-cold-start первый раз может быть 5-8 сек — это OK.

## Что зафиксировать по итогам

После каждой VM записать в release-journal:

1. Версия Windows, разрешение, железо (RAM/CPU).
2. Статус SmartScreen (показался / не показался).
3. Сколько секунд занял первый запуск `Tezis.exe`.
4. Прошёл ли Ollama wizard end-to-end (с моделью).
5. Прошёл ли `state-exam-full` цикл (attempt → review → FSRS запись).
6. Список найденных багов с приоритетом: **blocker** (не позволяет пользоваться) / **major** (раздражает) / **cosmetic** (косяк в UX).

## Release gate

**Разрешено рассылать classmates**, если на обеих VM:
- 0 blocker-багов
- ≤ 2 major-бага (и оба с workaround в README)
- FSRS persistence работает
- Ollama wizard завершается без ручного вмешательства (или дана чёткая инструкция на фоллбек в README)
- 10-шаговая инструкция в `README_classmates.md` исполнима классмейтом-непрограммистом

Если есть blocker — фикс в ветке `flet-migration`, пересборка Zip, повторный прогон checklist.
