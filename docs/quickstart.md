# Быстрый старт

Документ описывает текущую Flet-версию приложения из этого репозитория.

## 1. Установка зависимостей

```powershell
python -m pip install -r requirements.txt
```

## 2. Запуск

```powershell
python -m ui_flet.main
```

## 3. Что вы увидите

В приложении есть пять пользовательских экранов:

- `Onboarding`
- `Дневник`
- `Билеты`
- `Тренировка`
- `Настройки`

Если профиля ещё нет, приложение откроет `Onboarding`. Иначе корневой маршрут это `Дневник`.

## 4. Базовый цикл подготовки

1. Откройте `Билеты`.
2. Найдите нужный билет по номеру, тексту, разделу или сложности.
3. Откройте его карточку.
4. Пройдите режимы в порядке:
   `reading` → `plan` → `cloze` → `active-recall` → `state-exam-full` → `review`.
5. Возвращайтесь в `Дневник` и работайте по очереди на сегодня.

## 5. Что умеют экраны

`Дневник`
- показывает утреннюю очередь, дневную ленту попыток и итог дня
- даёт короткий вход в тренировку на сегодня

`Билеты`
- показывает все 208 билетов
- умеет поиск
- умеет фильтр по разделу и сложности
- на широком экране показывает очередь повторений и прогресс рядом с каталогом

`Тренировка`
- работает только с шестью режимами, встроенными в `ui_flet/workspaces/`
- для свободного ответа спрашивает уверенность до проверки

`Настройки`
- переключают тему
- переключают полноэкранный и оконный режим
- меняют шрифтовой пресет
- проверяют доступность `Ollama`

## 6. Ollama

`Ollama` не обязательна. Без неё приложение остаётся рабочим и использует упрощённую рецензию по ключевым словам.

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\install_ollama_wizard.ps1
```

macOS:

```bash
bash scripts/setup_ollama_macos.sh
```

## 7. Проверка

```powershell
python -m pytest -q
```

## 8. Сборка

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_flet_exe.ps1
```

macOS:

```bash
bash scripts/build_mac_app.sh python3 data/state_exam_public_admin_demo.db
```
