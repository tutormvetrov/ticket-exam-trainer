# Product Spec

## Назначение

Приложение предназначено для локальной подготовки к обычным экзаменам и госэкзамену на Windows. Базовая единица работы не билет как строка, а билет как карта знаний с атомами, навыками, упражнениями, weak areas и очередью повторения. Отдельно в продукт уже встроена первая рабочая DLC-вертикаль для подготовки к защите магистерской.

## Текущее состояние продукта

Уже реализовано:

1. Импорт учебных материалов из `DOCX` и `PDF` в основном exam-flow.
2. Превращение текста в структурированные билеты и внутреннюю карту знаний.
3. Оценка владения материалом по микронавыкам.
4. Адаптивное повторение слабых мест.
5. Локальная интеграция с `Ollama`.
6. Профиль ответа `Госэкзамен` с отдельным scoring и статистикой.
7. Первая рабочая DLC-вертикаль `Защита DLC`.

Остаётся целевым состоянием:

1. Полноценный `import preview` до старта импорта.
2. Дополнительный polish поверх уже закрытого Windows click-audit, если появятся новые UI-ветки или режимы.
3. Расширение DLC до полного ранее согласованного объёма.
4. Подтверждённый runtime QA на реальном macOS-устройстве.

## Supported User Training Modes

В текущем UI пользователю доступны 7 отдельных режимов:

- `reading`
- `active-recall`
- `cloze`
- `matching`
- `plan`
- `mini-exam`
- `state-exam-full`

Это и есть основной пользовательский truth-source для training UI.

## Internal Exercise Types

Во внутреннем движке и шаблонах упражнений есть более широкая taxonomy:

- `answer_skeleton`
- `structure_reconstruction`
- `atom_recall`
- `semantic_cloze`
- `odd_thesis`
- `oral_short`
- `oral_full`
- `examiner_followup`
- `weak_area_repeat`
- `cross_ticket_repeat`

Важно:

- эти типы реально существуют в доменной модели и exercise templates
- не все из них выведены в UI как отдельные пользовательские workspace
- продуктовая документация не должна описывать их как отдельные доступные режимы, если пользователь не видит их напрямую

## Import Scope

Основной продукт:

- поддерживает `DOCX` и `PDF`
- показывает progress по этапам
- сохраняет частичный результат без декоративного success

DLC defense-flow:

- дополнительно поддерживает `PPTX`, `TXT`, `MD`
- использует отдельный import path для материалов защиты

## DLC Status

`Защита DLC` уже встроен в продукт как сильный локальный модуль подготовки к защите.

Сейчас реально доступны:

- paywall
- локальная активация по ключу
- создание проектов
- импорт материалов
- построение dossier
- outline доклада
- storyboard слайдов
- вопросы комиссии
- mock defense

Что ещё не стоит считать полностью закрытым:

- DLC не является завершённым коммерческим модулем
- не все ранее планировавшиеся defense workflow выведены как самостоятельные пользовательские режимы
- в продукте нет встроенного billing flow

## Hard Constraints

1. Никаких облачных LLM.
2. Никаких декоративных статусов подключения.
3. Никаких немаркированных заглушек.
4. Никакой подмены реальных данных demo-данными без явной маркировки.
5. Любой неполный статус должен формулироваться как неполный, а не как завершённый.
