# Быстрый старт

Этот сценарий рассчитан на пользователя, который получил готовую Windows-сборку или запускает проект из исходников после установки зависимостей из `requirements.txt`.

## 1. Запустите приложение

Готовая Windows-сборка из GitHub Releases:

```text
Tezis\Tezis.exe
```

Запуск из исходников:

```powershell
python -m pip install -r requirements.txt
python main.py
```

## 2. Проверьте локальный ИИ

Откройте `Настройки -> Ollama`.

![Настройки Ollama](screenshots/settings.png)

Что сделать:
1. Нажать `Проверить соединение`.
2. Если Ollama ещё не настроен, нажать `Автонастройка Ollama`.
3. После завершения снова нажать `Проверить соединение`.

Считаем всё готовым, когда:
- `Endpoint` отвечает
- найдена `qwen3:8b` или совместимая локальная `Qwen`-модель
- показано время отклика
- статус подтверждается не только цветом, но и диагностикой

## 3. Импортируйте билеты

Откройте `Импорт документов`.

![Импорт](screenshots/import.png)

Рекомендуемый сценарий:
- один большой `DOCX`

Также поддерживается:
- `PDF`

Что важно знать заранее:
- отдельного `import preview` перед запуском импорта сейчас нет
- во время импорта приложение показывает progress по этапам
- после импорта появится summary с warning и handoff в библиотеку, тренировку и статистику

## 4. Откройте библиотеку

После импорта откройте `Библиотеку`.

![Библиотека](screenshots/library.png)

Что проверить:
- документ появился в списке
- справа видны разделы и билеты
- статистика не пустая

## 5. Перейдите к тренировке

Откройте `Тренировку`.

![Тренировка](screenshots/training.png)

В текущем UI доступны 7 режимов:
- `reading`
- `active-recall`
- `cloze`
- `matching`
- `plan`
- `mini-exam`
- `state-exam-full`

Минимальный рабочий маршрут:
1. выбрать билет из очереди
2. выбрать режим
3. написать ответ
4. нажать `Проверить ответ`
5. посмотреть feedback и follow-up вопросы

## 6. Смотрите статистику

Откройте `Статистику`.

![Статистика](screenshots/statistics.png)

Там видно:
- средний результат
- слабые места
- последние сессии
- сильные и слабые стороны по навыкам

Для билетов с профилем `Госэкзамен` доступен отдельный block-level профиль готовности.

## 7. Что уже есть, а что нет

Есть:
- рабочий основной сценарий `проверка Ollama -> импорт -> тренировка -> статистика`
- прямая проверка Ollama
- сохранение данных в SQLite
- профиль `Госэкзамен`
- отдельный paywalled workspace `Защита DLC`

Что уже умеет DLC:
- активация по ключу
- создание проекта защиты
- импорт материалов
- dossier
- outline
- storyboard
- mock defense

Что по DLC пока не стоит обещать без оговорки:
- это сильный локальный DLC-модуль с manual activation
- для него тоже нужен локальный `Ollama`
- без активации DLC недоступен
- встроенного billing flow нет

## 8. Repo / Release Truth

Если вы работаете не только как пользователь, но и как сборщик релиза:
- root `README.md`, `docs`, `scripts`, код и тесты являются редактируемыми исходниками
- `build` считается временным build-artifact
- `dist` считается итоговым output пакета
- содержимое `dist/Tezis/*` не редактируется вручную, а пересобирается через `scripts/build_exe.ps1`
- packaged build содержит `build_info.json`, а интерфейс показывает текущую версию и commit
- быстрый release smoke выполняется через `scripts/release_smoke.ps1`

## 9. Если что-то не работает

### Ollama не отвечает

Проверьте:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\check_ollama.ps1
```

### Нужно полностью поднять локальную среду

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup_ollama_windows.ps1
```

### Импорт не прошёл

Проверьте:
- что файл действительно `.docx` или `.pdf`
- что файл не битый
- что в summary нет ошибки формата

## 10. macOS

Поддержка macOS в репозитории подготовлена, но финальный ручной smoke-run нужно выполнять на реальном Mac.

Запуск из исходников:

```bash
python3 main.py
```

Подготовка локального Ollama:

```bash
bash scripts/setup_ollama_macos.sh
bash scripts/check_ollama_macos.sh
```

Сборка `.app` на реальном Mac:

```bash
bash scripts/build_mac_app.sh
```

Если macOS блокирует unsigned build:

```bash
xattr -dr com.apple.quarantine dist/Tezis.app
```
